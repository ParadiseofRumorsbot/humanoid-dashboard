## 0. 한 줄 요약

**탭 11개를 5개로 줄인다. 내용은 한 글자도 안 바꾸고 "집만 옮긴다". 첫 탭에 전체 목차를 달아 어디든 한 번의 클릭으로 간다. 기술은 [전체 그림 → 소프트웨어 → 하드웨어] 순서로 배치한다.**

```
지금 (11탭)                                     나중 (5탭)
─────────────────────────────                  ─────────────────────────
프레임워크                                      ① 프레임워크 (왜 투자하나 + 병목 지도 + 전체 목차)
TAM 시뮬레이터      ┐                           ② 시장·경쟁 (얼마나 큰 시장이고 누가 이기고 있나)
기업별 TAM          ├─ 시장 얘기 3곳             ③ 기술 (어떻게 작동하나)
경쟁 구도           ┘                              ├ 서브탭 1: 전체 그림
부품 기술           ┐                              ├ 서브탭 2: 소프트웨어(두뇌)
휴머노이드 해부     ├─ 기술 얘기 4곳               └ 서브탭 3: 하드웨어(몸)
Physical AI         │                           ④ 밸류체인 (누가 만들고 누가 돈 버나)
기술 설명           ┘                              ├ 서브탭: KR  └ 서브탭: US
US 밸류체인         ┐─ 밸류체인 2곳              ⑤ 밸류에이션 (그래서 얼마짜리 회사인가)
KR 밸류체인         ┘
밸류에이션
```

읽는 순서 자체가 스토리다: **왜(①) → 얼마나(②) → 어떻게(③) → 누가(④) → 얼마(⑤)**.

## 2. 🔒 콘텐츠 보존 원칙 — 내용은 한 글자도 임의로 바꾸지 않는다 (v2 신설, 최우선 규칙)

### 허용되는 것 (이것만 한다)
1. **코드째 이동**: 기존 카드(HTML 블록·SVG·시뮬레이터 JS)를 복사-붙여넣기로 새 파일에 옮긴다. 문구·수치·출처·[추정] 태그·색·이모지 전부 그대로.
2. **신규 작성**: nav·서브탭·앵커 id·목차·"→ 더 보기" 연결문·새 시각자료·병목 지도 같은 **새로 추가되는 것만** 새로 쓴다. (12살 스타일 가이드는 이 신규 텍스트에만 적용)
3. **계획된 중복 대체**: §4 매핑표에 미리 명시된 중복 문단만 "요약 1줄 + 정본 링크"로 교체. 매핑표에 없는 삭제·축소는 금지.

### 금지되는 것

## 3. 최종 모습 — 5탭 상세 설계

### 탭 ① 프레임워크 — "왜 지금 휴머노이드인가" (파일: framework.html 유지·보강)

- 대상 독자: 아무것도 모르는 사람이 **처음 여는 페이지**.
- 12살 설명: "이 산업이 왜 큰일인지, 뭐가 막혀 있는지, 어디에 돈을 걸어야 하는지 요약한 표지 + 전체 지도."
- 페이지 구성(위→아래):

**①-A. 📑 전체 목차 (v2 신설, 사용자 요청 ⑥)**
  - 페이지 최상단에 접이식 목차 카드. 5탭 × 서브탭 × 주요 섹션의 **3단 트리**.
  - 항목 클릭 → 해당 페이지의 섹션 앵커로 바로 이동 (`market.html#tam-sim` 식).
  - 전제 작업: 모든 새 페이지의 주요 섹션에 `id` 부여 (각 단계 커밋에 포함, C6에서 전수 점검).
  - 형태 예시:
    ```
    ② 시장·경쟁          ③ 기술                    ④ 밸류체인
      · TAM 시뮬레이터      · 전체 그림: 폐루프        · KR: 종목별 투자 포인트
      · 기업별 TAM          · SW: VLA / 월드모델       · US: 계층별 분류
      · 경쟁 현황            · HW: 해부 / 부품 데이터   ⑤ 밸류에이션 · 3단 워터폴
    ```

**①-B. 관통 메시지 3개** (해부 탭에서 공유): 열이 최대 적 / HW는 AI의 데이터 도구 / 자동차 공급망이 답 + "열을 안 만드는 해결책 4종 패키지" 요약(정본은 해부 탭, 링크)

**①-C. 🚧 휴머노이드의 병목 지도 (v2 신설, 사용자 요청 ①)** — "뭐가 막혀서 아직 로봇이 우리 집에 없나"
  - 12살 설명: "고속도로에 차가 막히는 지점 4곳을 표시한 지도."
  - 병목 4대 축, 각각 [한 줄 정의 + 핵심 수치 배지 + 정본 링크]:
    | 병목 | 한 줄 (비유 먼저) | 핵심 수치 | 정본 |
    |---|---|---|---|
    | 🌡️ 열 | 로봇은 일 조금 하는 난로 — 8시간을 못 버틴다 | 입력의 ~90% 열, 전완 병목 9/10 [자체추정] | ③-3 해부 07 |
    | 🧠 데이터 | 두뇌 학습에 먹일 "몸 경험 데이터"가 없다 | 100대 vs 10,000대 격차, 수집 비용 | ③-2 SW |
    | 💰 원가/양산 | 액추에이터 24개 = 자동차 한 대 값 | $1,000 → $100 목표 | ② BOM·③-3 |
    | 🔩 공급 부품 | 정밀 감속기·롤러스크류는 만들 수 있는 회사가 적다 | 공급 병목 히트맵 | ② 히트맵·④ |
  - 구현: framework 기존 "물리 레이어 병목 — 5대 수혜 영역" 카드를 **무수정 유지**하고, 그 위에 이 병목 지도(신규)를 추가. 둘은 "무엇이 막혔나(신규) → 그래서 누가 수혜인가(기존)"로 자연 연결.

**①-D. 기존 framework 콘텐츠 전부 무수정 유지**: 핵심 투자 명제, 부품 수 급감, 상용화 6조건, 정부 3M, NVIDIA 레이어 케이크, 물리 레이어 병목 5대, 칩 vs 액추에이터, 전력 수요 규모 비교

**①-E. competition에서 이사(무수정)**: 한국 소버린 운영 OS 포지션, NVIDIA vs Google 풀스택 전쟁
